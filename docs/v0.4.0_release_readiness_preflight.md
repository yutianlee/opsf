# v0.4.0 Release Readiness Preflight

Date: 2026-05-14

This document records a documentation-only preflight after the v0.4.0
positive-real gamma-ratio and beta method plan landed. It does not authorize a
release, create or move a tag, bump the package version, change workflow files,
publish to PyPI or TestPyPI, update smoke-test pins, or promote any explicit
method into default dispatch.

The method implementation closeout is recorded in
[`v0.4.0_method_closeout.md`](v0.4.0_method_closeout.md). The development plan
is recorded in [`release-0.4.0.md`](release-0.4.0.md).

## Preflight Decision

The v0.4.0 explicit positive-real method scope is implementation-complete for
the planned `loggamma_ratio`, `gamma_ratio`, and `beta` methods. A v0.4.0
release remains a separate release action and is still gated by release policy
and explicit current-task authorization.

Before any release action, a separate release-preparation change should record
the exact package version, release tag, GitHub release type, PyPI/TestPyPI
choices, post-release smoke plan, changelog entry, release checklist updates,
and workflow default updates allowed for that release.

## Implemented v0.4.0 Method Scope

| Explicit method | Certificate scope | Implementation PR | Domain |
| --- | --- | --- | --- |
| `loggamma_ratio(a, b, mode="certified", method="stirling_diff", dps=...)` | `loggamma_ratio_positive_real_stirling_diff` | PR #122 | finite real `a >= 20`, `b >= 20` |
| `gamma_ratio(a, b, mode="certified", method="stirling_ratio", dps=...)` | `gamma_ratio_positive_real_stirling_ratio` | PR #123 | finite real `a >= 20`, `b >= 20` |
| `beta(a, b, mode="certified", method="stirling_beta", dps=...)` | `beta_positive_real_stirling_beta` | PR #124 | finite real `a >= 20`, `b >= 20` |

The v0.3 explicit positive-real `loggamma`, `gamma`, and `rgamma` methods
remain limited to finite real `x >= 20`.

## Dispatch Guardrails

The v0.4.0 method work preserves the direct Arb defaults. The new methods are
selected only through their exact explicit method names in certified mode.

The following behavior remains unchanged:

- omitted `method` behavior;
- `method="auto"` behavior;
- default certified dispatch;
- `method="arb"` direct Arb selection; and
- existing direct Arb paths for `loggamma_ratio`, `gamma_ratio`, and `beta`.

## Release Boundary

No release action is authorized by this preflight. Per project policy, a
release task must explicitly name:

- repository name;
- release tag;
- package version;
- whether the GitHub release is a prerelease or normal release;
- whether PyPI publishing is allowed;
- whether TestPyPI publishing is allowed or skipped; and
- whether post-release `pypi-smoke` should be run.

Until those details are provided in a release task, no tag, GitHub release,
PyPI/TestPyPI publication, smoke-pin update, or post-release verification PR
should be started.

## Exclusions

This preflight does not add or claim:

- complex gamma-ratio Stirling certification;
- complex beta Stirling certification;
- reflection-formula certification;
- near-pole behavior;
- simultaneous-pole limiting values;
- default-dispatch promotion;
- parabolic-cylinder promotion;
- new public wrappers;
- workflow changes;
- version changes;
- package publication; or
- broad package-wide certification.

## Future Release-Preparation Work

A later release-preparation PR, under explicit release authorization, should:

- update version-bearing metadata for the authorized package version;
- update release notes, changelog, and release checklist entries;
- decide TestPyPI staging under [`release_policy.md`](release_policy.md);
- run `python scripts/check_release_version.py <release-tag>`;
- run the full local validation gate;
- keep `pypi-smoke.yml` pinned to the last real PyPI release until the new
  version has actually published and passed smoke verification; and
- avoid runtime, certification-scope, and default-dispatch changes during the
  release-preparation step.
